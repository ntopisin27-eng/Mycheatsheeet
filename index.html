Шпаргалка по Git

Команды Git
Команда Описание
git init - Создает новый локальный репозиторий в текущей папке.
git clone url - Скачивает удаленный репозиторий по указанному URL-адресу и создает его локальную копию.
git add file - Добавляет указанный файл в область подготовленных файлов (staging area), отслеживая его изменения для следующего коммита.
git add . - Добавляет все измененные и новые файлы в текущей папке и во всех вложенных в область подготовленных файлов.
git commit -m "comment" - Фиксирует (сохраняет) все изменения, находящиеся в области подготовленных файлов, и создает из них новый коммит (точку в истории) с указанным комментарием comment.
git commit -a -m "comment" - Автоматически добавляет все отслеживаемые измененные файлы в область подготовленных файлов (но не новые, неизвестные Git'у файлы) и сразу создает коммит с комментарием comment.td
git push url - Отправляет (загружает) все коммиты из вашей локальной ветки в удаленный репозиторий по указанному URL-адресу.
git status - Показывает текущее состояние рабочей директории и области подготовленных файлов: какие файлы изменены, добавлены, готовы к коммиту или не отслеживаются Git'ом.
git log - Выводит полную историю коммитов текущей ветки: автора, дату, уникальный хеш и комментарий для каждого коммита.
git branch branch_name - Создает новую ветку с именем branch_name. Сама ветка создается, но вы автоматически на нее не переключаетесь.
git checkout branch_name - Переключает вас на уже существующую ветку с именем branch_name. Рабочая директория обновится до состояния этой ветки.
git checkout -b branch_name - Создает новую ветку branch_name и сразу переключает вас на нее. Это сокращение для двух последовательных команд: git branch branch_name и git checkout branch_name.
git remote - Позволяет управлять списком удаленных репозиториев, связанных с вашим локальным. Например, git remote -v покажет URL-адреса для получения (fetch) и отправки (push) изменений.
